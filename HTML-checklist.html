HTML Checklist for Pages and Sites

Use this document to check web pages and web sites for Section 508 (accessibility) compliance. For assistance, contact Local person.

2019 testing notes

The content of this checklist is based on the 508-WCAG 2.0 requirements in effect since January 18, 2018, as interpreted in HHS's Digital Communications Division (DCD) checklist. Last checked for changes May 2019 (when HHS's last reviewed date was May 6, 2019). The order of testing here has been optimized for subject matter experts who only control the content area of their pages, with the most-commonly found problems first. Site-related checks are consolidated into a new section. All HHS checkpoints remain. Some procedural and remediation information has been added. If ‘No’ is answered to any item, meaning something is not Perceivable, Operable, Understandable, or Robust, then the page or site is not 508 compliant. CSS files need to be validated at W3C. NVDA screen reader users: A recommendation for sighted users is to change the default from Insert to Caps Lock. So the instructions below to open the NVDA Elements List is described as Caps Lock-F7. Important remediation resource it would be useful to be familiar with: How to Meet WCAG 2 Quick Reference, filtered to AA

Getting started

Use Chrome with the axe and ANDI add-ons for most checks, but use Firefox when testing with the NVDA screen reader, and use Firefox and its Web Developer extension (gear in menu bar) if you want to outline table cells, display alt text on top of the images they belong to, etc. More about testing tools and procedures. When work will be turned back to the page/site owner, use the check boxes (starting with the first checkbox below) and email the report, which will ask the page/site owner to finish running the whole checklist. Between documents, reload this page to clear checkboxes.

  • WCAG 2.0 Success Criterion 1.3.1
  • Means documents authored or otherwise controlled by our organization such as PDF files, Word files, etc.

  • WCAG 2.0 Success Criterion 4.1.1
  • Use the W3C HTML validator to check whether any opening tags, closing tags or attributes are malformed.

Color

  • WCAG 2.0 Success Criterion 1.4.1
  • Problem example: A chart uses green for good and red for bad with no other way to tell them apart.

  • WCAG 2.0 Success Criterion 1.4.3 (Size stipulation not considered by HHS)
  • In Chrome, open the ANDI tool and select "color contrast" from the drop-down, Use the arrow-type buttons to move through the questionable items (if any). Or, run WAVE and click the Contrast tab in the results panel.
  • axe: Text elements must have sufficient color contrast against the background
  • G18: Ensuring that a contrast ratio of at least 4.5:1 exists between text (and images of text) and background behind the text
  • W3C Understanding WCAG 2.0 Contrast (Minimum) Understanding SC 1.4.3

Tables
